# 问题

现有**幻觉检测**方法多针对独立大语言模型，无法适配检索增强生成（RAG）系统需与检索证据一致的需求。人工标注一般提供的是“回答级” 标签（整体是否为幻觉），对于事实细粒度幻觉缺乏关注。

# 方案

对 RAG 生成答案的原子事实进行突变，例如：

* **同义词蜕变关系** ：对原始事实中的关键术语（如 “安全”“允许”）进行同义词替换，生成语义完全等价的事实变体（例如将 “布洛芬在孕期安全” 变为 “布洛芬在妊娠期无害”）
* **反义词蜕变关系** ：对原始事实中的关键术语进行反义词替换或否定，生成语义完全相反的事实变体（例如将 “布洛芬在孕期安全” 变为 “布洛芬在孕期危险”）

对每个事实的变体计算事实得分（通过 LLM 判断是否符合事实)，通过对正向和反向表述的计算，来验证是否出现幻觉。

# 实验

* 在标注的幻觉数据上测试MataRAG 对幻觉识别的精确率，
* 案例分析：
  * 用户查询 “孕妇能否用布洛芬治背痛”，RAG 生成回答 “布洛芬孕期全程安全”，但上下文 C 明确 “孕晚期禁用布洛芬”。MetaRAG 将回答分解为 “布洛芬可治背痛”“布洛芬孕期全程安全” 两个事实，其中 “孕期全程安全” 的事实得分 **\(S_i=0.92\)**（高幻觉风险），触发 “高亮风险片段 + 强制引用 + 人工审核” 的防护措施；
* 消融实验
  * 变异的数量
  * 事实分解模型
  * 变异温度
  * 验证器模型
